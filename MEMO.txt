//--------------- JavaDB Tips ------------------------

Database ファイルは Glassfish の INSTALL_DIR/database の中。
 /usr2/local/glassfish/databases
   dnd_db

・制約の削除
  ALTERL TABLE CHARACTER_RECORD DROP CONSTRAINT "制約名";
・外部キー制約の追加
  ALTERL TABLE CHARACTER_RECORD ADD CONSTRAINT "制約名" FOREIGN KEY (CHARACTER_ID) REFERENCES CHARACTER_RECORD(ID) ON DELETE CASCADE;

	"KNOWLEDGE_NOBILITY_MISC" INT DEFAULT 0, 貴族
	"KNOWLEDGE_ARCH_ENG_MISC" INT DEFAULT 0, 建築工学
	"KNOWLEDGE_THE_PLANES_MISC" INT DEFAULT 0, ？？ 次元界
	"KNOWLEDGE_NATURE_MISC" INT DEFAULT 0,  自然
	"KNOWLEDGE_RELIGION_MISC" INT DEFAULT 0, 宗教
	"KNOWLEDGE_ARCANA_MISC" INT DEFAULT 0, 神秘
	"KNOWLEDGE_DUNGEON_MISC" INT DEFAULT 0,  ダンジョン
	"KNOWLEDGE_LOCAL_MISC" INT DEFAULT 0, 地域
	"KNOWLEDGE_GEOGRAPHY_MISC" INT DEFAULT 0,  地理
	"KNOWLEDGE_HISTORY_MISC" INT DEFAULT 0, 歴史

・DDL の生成
  dblook コマンドを使う。以下は dnd_db_ddl というファイルに DDL を書き出す例
  C:\Users\ka78231>"C:\Program Files\glassfish-v2ur2\javadb\bin\dblook" -d "jdbc:derby://localhost:1527/dnd_db;user=app;password=app" -o c:\Users\ka78231\Documents\dnd_db_ddl

・s10 の JavaDBの起動
 # asadmin start-database

・s10 の JavaDB への接続 URL
  jdbc:derby://s10.whiteboard.ne.jp:1527/dnd_db

・ijコマンド(sqlpulus みたいなもの？）
  # ij
  リモートへ接続
  ij> connect 'jdbc:derby://s10.whiteboard.ne.jp:1527/dnd_db;;user=app;password=app;create=true;'
  ローカルへ接続k
  ij> connect 'jdbc:derby://localhost:1527/dnd_db'
